Semana 3 – Desarrollo, pruebas y entrega (25 al 29 de agosto)

Objetivo: Completar las funcionalidades esenciales del sistema, integrando todas las partes en un flujo de uso estable; realizar pruebas funcionales y end-to-end para garantizar su correcto funcionamiento; aplicar mejoras de estilo y experiencia de usuario; documentar el proyecto y publicarlo, dejándolo listo para la sustentación.

Lunes:
Ampliación de funcionalidades. Implementar funciones para actualizar y eliminar datos, aplicar estilos y validaciones.

Martes:
Integración y mejoras. Integrar todas las partes del proyecto, optimizar flujo de datos y mejorar la experiencia de usuario. 

Miercoles:
Testing funcional + Ajustes finales. Probar todas las funciones, corregir errores y optimizar el rendimiento.

Jueves:
Revisión + Verificación de alcance (Scope). Demostración del funcionamiento completo del sistema y cierre del backlog. Revisión con todo el equipo de los objetivos cumplidos, funcionalidades entregadas y feedback del cliente/stakeholder.

Viernes:
Testing e2e + Deploy + Documentación. Pruebas end-to-end, despliegue del proyecto y finalización de la documentación.

Entregables Semana 2:
• Proyecto funcional completo
• Link público al proyecto desplegado (Netlify, GitHub Pages, etc)
• Documentación en README (tecnologías, instrucciones, autores)
• Documento técnico


Recuerda que el objetivo final es entregar algo 100% funcional, no a medias (MVP)
